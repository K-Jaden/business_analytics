## 1. 한 줄 요약

**"리셀 시장(스니커즈·카드·레고)에서 미디어 채널(구글 트렌드·뉴스·유튜브)이 가격에 선행하는가,
그리고 선행 채널만 써도 전채널만큼 예측할 수 있는가를 Granger + XGBoost + DM으로 검증했다."**

## 2. 논문 전체 흐름 (처음 → 끝)

```
서론 → 관련연구 → 데이터·방법론 → 결과(4절) → 논의 → 결론
```

### 서론
- 리셀 시장 규모·성장 배경
- 기존 연구: 주가·암호화폐에서 미디어 선행성은 검증됨 → 리셀 자산은 공백
- 이 논문의 기여: 3자산·5채널 비교, Granger→XGBoost 2단계 검증

### 관련연구
- Granger 인과성과 금융 예측 (주요 선행 연구)
- 미디어 감성과 자산 가격 (FinBERT, GDELT 활용 연구)
- XGBoost + SHAP의 금융 시계열 적용 사례

### 데이터·방법론
- **수집 대상:** 15개 아이템 × 48개월 (2022-01 ~ 2025-12)
- **5개 채널:** Google Trends / 뉴스량 / 뉴스감성 / YT조회수 / YT댓글감성
- **3단계 분석:**
  - STEP 1: ADF 정상성 검정 → 비정상 시계열 차분
  - STEP 2: Granger 인과 검정 (15회, BH 보정)
  - STEP 3: XGBoost Model A vs B + SHAP + DM 검정

### 결과절 (현재 완성된 부분)

| 절 | 내용 | 완성 여부 |
|----|------|--------|
| §4.1 ADF | 18개 시계열 정상성 검정 결과 | ✅ 완성 |
| §4.2 RQ1 | Granger 15회 결과 표 | ✅ 완성 (단, BH 미적용 — 아래 논의) |
| §4.3 RQ2 | Jaccard 유사도 표 | ✅ 완성 |
| §4.4 RQ3 | DM 검정 결과 + SHAP 요약 | ✅ 완성 |

### 논의절 ← **미작성, 팀 합의 필요**
- SHAP-Granger 불일치 해석
- jordan1·falcon에서 Model A 우수한 원인
- 자산별 지배 채널 차이 해석
- 한계점(limitation)

### 결론절 ← **미작성**
- RQ1·2·3 종합 답변
- 실무적 시사점
- 향후 연구 방향

## 4. 팀이 결정해야 할 사항 (3가지)

### 결정 ① BH 보정 적용 여부

**상황:**
- 원래 설계에 BH 보정 포함이었으나 실수로 미적용
- 적용 시: sneakers CH4(p=0.035)가 유의에서 탈락, SIGNIFICANT 2개만 남음

**선택지:**
- **A) 적용** — 학술적으로 맞음. 결과가 더 보수적이 됨. 논문 신뢰도 올라감.
- **B) 미적용 + 명시** — "탐색적 연구(exploratory)"임을 밝히고 raw p 그대로 씀. 일부 저널에서 허용.

**추천: A (BH 적용)** — 15회 동시 검정에서 보정 안 하면 심사 때 반드시 지적받음

---

### 결정 ② SHAP-Granger 불일치를 어떻게 쓸 것인가

**상황:**
- Granger SIGNIFICANT인 CH3(뉴스감성)의 SHAP = 0.011 (채널 중 최하위)
- Granger NOT_SIG인 CH5(YT댓글감성)의 SHAP = 0.124 (채널 중 2위)
- 이 불일치가 논문 전체에서 가장 설명이 필요한 지점

**왜 불일치하는가:**
```
Granger = "이 채널 혼자, 가격 과거값만 놓고 봤을 때 선행하는가" (선형, 단독)
SHAP    = "다른 모든 변수와 함께 있을 때 이 채널의 기여도" (비선형, 다변수)

→ 가격 모멘텀(price_vs_ma3, price_chg_lag1)이 분산의 대부분을 설명해버리면
  Granger에서 유의했던 채널도 SHAP에서는 기여할 게 없어짐
```

**선택지:**
- **A) 논의절에서 설명하고 넘어감** — 한 단락으로 정리. 추가 실험 없음.
- **B) 채널만 SHAP 별도 계산** — 가격 변수 제외 모델로 채널 간 상대 순위 확인. 하지만 연구 설계 바뀜.

**추천: A** — 불일치 자체가 발견이고, 설명 가능한 범위 안에 있음

---

### 결정 ③ DM 검정이 회귀 기반인 것을 어떻게 서술할 것인가

**상황:**
- 메인 XGBoost는 분류 (price_direction, AUC)
- DM 검정은 회귀 (mean_price, RMSE) 기반으로 구현됨
- 두 태스크가 다름

**선택지:**
- **A) 현재 그대로 + 각주에 명시** — "DM 검정은 가격 수준 예측 오차(RMSE) 기반으로 수행됨"
- **B) 분류 기반 DM으로 재구현** — 예측 확률 vs 실제 0/1의 Brier score로 DM. 추가 작업 필요.

**추천: A** — DM 검정은 원래 point forecast 비교 도구라 회귀가 더 자연스럽고, 각주로 충분히 정당화됨

## 7. ⚠️ 핵심 구조 문제 — BH 보정과 RQ3 연결성

### 문제 상황

BH 보정 적용 시 자산별 유의 채널:

| 자산 | BH 적용 후 유의 채널 | Model B 구성 |
|------|------------------|------------|
| sneakers | CH3만 (p=0.0007) | 가격변수 4개 + CH3 1개 |
| cards | CH1만 (p=0.006) | 가격변수 4개 + CH1 1개 |
| **lego** | **없음** | **가격변수만 ← 문제** |

레고는 Granger 유의 채널이 없으므로 Model B = 가격 변수만 남음.
"Granger로 채널 선별 → XGBoost로 검증"이라는 논문 핵심 파이프라인이 레고에서 끊김.
즉 **BH 적용하면 RQ3로 자연스럽게 연결되지 않는다.**

### 선택지 3가지

**A) BH 적용 + 레고 제외**
- 스니커즈·카드만으로 RQ1~3 완결
- 레고는 "유의 채널 미발견" 별도 보고
- 단점: 표본이 2자산으로 줄어 일반화 약해짐

**B) BH 미적용 + "탐색적 연구(exploratory)" 명시 ← 추천**
- raw p 그대로 사용, 논문 서두에 "exploratory study" 포지셔닝
- BH 결과는 부록에 robustness check으로 첨부
- 48개월·3자산 소표본이라는 데이터 규모가 탐색적 연구에 적합
- 탐색적 금융 연구에서 널리 사용되는 방식

**C) BH 기준 완화 (q=0.10)**
- sneakers CH4(p=0.035)까지 통과
- lego CH1(p=0.054)은 여전히 탈락 → 레고 문제 해결 안 됨

### 팀 합의 포인트

> **"이 논문을 confirmatory(가설 검증)으로 볼 것인가, exploratory(탐색적)으로 볼 것인가?"**
>
> - Confirmatory → BH 필수 → 레고 RQ3 제외 → 2자산 논문
> - Exploratory → BH 부록 → 3자산 유지 → 더 풍부한 결과
