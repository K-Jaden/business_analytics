# 나중에 할 것 — 논문 보강 검토 사항

> 받은 제안에 대한 평가 및 우선순위 (2026-05-21)
> 현재 상태: KCI 보강 완료 83.5/100, 22쪽

---

## 받은 제안 4가지 (원문)

### 2. 태스크 불일치 해결 → '분류(Classification)'로 완벽 통일
- **기존 문제:** 모형 학습과 SHAP은 '가격 방향성 분류'로 해놓고, DM 검정은 '가격 수준 회귀'로 검증함
- **해결책:** DM 검정의 손실함수를 회귀의 제곱오차(MSE)가 아닌, 분류 모형의 예측 확률값을 이용한 브라이어 스코어(Brier Score)나 로그 손실(Log Loss)로 변경
- **구체적 수정:** XGBClassifier의 예측 확률 $p_t$와 실제 방향 타겟 $y_t(0 or 1)$ 사이의 오차 시계열 $e_t = (p_t - y_t)^2$로 DM 검정
- **효과:** "분류 확률에 대한 DM 검정은 결과 안정성이 낮다"는 변명조 각주 제거 가능

### 3. 정상성 통과 실패 변수 해결 → 'Toda-Yamamoto' 절차 도입
- **기존 문제:** 스니커즈의 Google Trends 변수가 1차 차분 후에도 정상성 미충족 → 가짜 회귀 우려
- **해결책:** Toda-Yamamoto (1995) 절차로 논증 구조 변경
- **방법:** 변수들의 최대 단위근 차수($d_{max}=1$ 혹은 $2$)를 구한 뒤, 최적 래그 $L$에 $d_{max}$만큼 추가($L + d_{max}$) → VAR 모형으로 인과성 검정
- **효과:** 비정상 시계열도 차분 없이 수준값 그대로 Granger 인과성 검정 가능

### 4. 분석 단위 불일치 해결 → '패널 Granger 인과성 검정' 수행
- **기존 문제:** Granger는 자산 평균(N=48)으로, XGBoost는 아이템 패널(N=705)로 따로 노는 문제
- **해결책:** Dumitrescu-Hurlin (2012) 이질적 패널 Granger 검정 실행
- **방법:** Python `linearmodels` 또는 `statsmodels` 패널 인과성 함수 사용 → 15개 아이템 패널 데이터 전체 묶어서 단일 패널 Granger 결과 도출
- **효과:** Granger와 ML의 분석 단위가 '아이템-월 패널'로 정합

### 5. 소표본 ML 남용 지적 방어 → 모형 다이어트 및 Baseline 추가
- **기존 문제:** 행 수 수십 개 수준인데 max_depth=6 → 과적합 위험
- **해결책:**
  - `max_depth`를 2~3으로 축소
  - `reg_alpha`(L1) / `reg_lambda`(L2) 강화
  - 로지스틱 회귀(Lasso/Ridge) 대조군 추가
- **효과:** "XGBoost가 단순 선형 모델보다 비선형 결합을 더 잘 포착" 스토리라인 추가

## 내 평가

### 2번 (Brier Score DM) — 강력 추천
가장 정당한 지적. 현재 학습/SHAP은 분류, DM은 회귀(MSE)인 비대칭은 심사자가 가장 먼저 찌를 약점. DM 검정 손실함수는 자유 선택 가능하므로 $e_t = (p_t - y_t)^2$ (Brier) 또는 log loss로 바꾸는 건 수학적으로 완전 정당.

- **리스크:** T=24/아이템에서 Brier score 차이가 매우 작아 DM 통계량 자체가 노이지해질 수 있음. 결과가 뒤집힐 가능성도 있음 (현재 sneakers_jordan1 p<0.001, falcon p=0.032의 유의성이 약화될 여지).
- 현재 각주에서 변명조로 쓴 "안정성 낮다"를 지우고 정면돌파한다는 발상은 맞음.

### 3번 (Toda-Yamamoto) — 추천하되 조건부
sneakers GT 한 변수의 비정상성 문제 해결책으로 TY는 표준적이고 합법적. 다만:

- **TY는 모든 변수에 일괄 적용**해야 함. 한 변수만 적용 불가. → 15개 검정 전체 재실행.
- N=48에서 $L + d_{max}$로 래그 증가 → 자유도 손실 → 검정력 하락. 현재 raw p<0.05 결과 중 일부(특히 marginal한 lego CH1 p=0.054)는 비유의로 바뀔 수 있음.
- **권장:** 본문은 기존 Granger 유지, **부록에 TY 결과 병기**. "정상성 우려 변수에 대한 robustness check"로 포지셔닝.

### 4번 (Dumitrescu-Hurlin 패널 Granger) — 가장 임팩트 크지만 가장 리스크 큼
현재 논문의 구조적 약점(자산 평균 N=48 vs 패널 N=720) 해결은 정확. DH는 이질성 허용 패널 인과성 검정으로, 자산 안에서 아이템 5개의 cross-sectional 정보 활용 가능. linearmodels로 구현도 단순.

- **리스크:** 결과가 자산-평균 Granger와 **상당히 다를 수 있음**. 현재 RQ2(자산별 채널 구성 다름) 결론의 핵심 논거가 자산-평균 Jaccard 비교인데, 패널로 가면 RQ2 재구성 필요할 수도.
- **결정 포인트:** 71.5→83.5점 보강을 최근 마친 상태에서 분석 단위를 바꾸면 사실상 4·5장 재작성. 학부/KCI minor revision 목적이면 **현재 구조 유지 + 한계점 단락 추가**가 비용 대비 안전. 박사논문/SSCI 노린다면 DH 도입 가치 있음.

### 5번 (모델 다이어트 + Baseline) — 부분 동의
- **max_depth=6 축소:** 현재 패널 N=720은 max_depth=6도 위험하진 않지만, T=24/아이템 시계열 구조 고려 시 3~4가 더 안전. 권장.
- **L1/L2 강화:** 권장. 비용 거의 0.
- **Logistic 베이스라인 추가:** 좋지만 현재 핵심 비교축이 Model A(전채널) vs Model B(Granger 선별)인데, LR 추가 시 비교 구조가 2×2가 되어 서사 복잡화. "전채널 LR vs 전채널 XGB" 한 줄만 표로 추가하는 정도가 적절.

## 종합 우선순위

| 순위 | 작업 | 비용 | 리스크 | 임팩트 |
|------|------|------|--------|--------|
| 1 | **2번 Brier DM** | 낮음 | 중 | 높음 (변명 각주 제거) |
| 2 | **5번 일부** (max_depth↓, L1/L2↑, LR 베이스라인 1줄) | 낮음 | 낮음 | 중 |
| 3 | **3번 TY를 부록 robustness로** | 중 | 중 | 중 |
| 4 | **4번 DH** | 높음 | 높음 | 매우 높음 (단, 학부/KCI 목적이면 보류) |

**결론:** 2번과 5번은 지금 바로 적용해도 손해 없음. 3번은 부록 robustness로 안전하게 추가. 4번은 논문 목적(KCI vs 상위 저널)에 따라 갈림.
