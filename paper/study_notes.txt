 발표자용 학습 노트
 "이 슬라이드가 뭘 의미하는가" — 본인이 완전히 이해하기 위한 안내서
═══════════════════════════════════════════════════════════════════════

[읽는 법]
 · 각 슬라이드마다 네 가지 블록으로 정리되어 있습니다.
   ① 한 줄 요약       : 이 슬라이드가 왜 있는가
   ② 쉬운 설명       : 핵심 개념을 일상 언어로
   ③ 숫자의 의미     : 화면에 보이는 수치가 뭘 말하는지
   ④ 질문 받으면     : 청중이 물을 가능성 + 답하는 키워드
 · 대본과 함께 보세요. 대본은 '어떻게 말할까', 이 노트는 '왜 그렇게 말하는가'.

[ S1. 표지 ]
═══════════════════════════════════════════════════════════════════════
① 한 줄 요약
   인사와 제목 소개. "내가 뭘 연구했는지" 한 문장으로 던지는 자리.

② 쉬운 설명
   리셀 시장 = 중고지만 한정판이라 정가보다 비싸게 거래되는 시장.
   '미디어가 가격을 미리 알려주나?' 가 핵심 질문이다.

③ 숫자의 의미
   2022.01–2025.12 (48개월) = 4년치 월별 데이터.
   15 items = 자산 3종 × 5개씩.

④ 질문 받으면
   - "왜 48개월이냐?" → StockX·PriceCharting에서 안정적으로 수집 가능한
     기간이 거기까지였다. 더 길게 가면 자료원·아이템 변경 이력이 누적.

[ S2. 연구 동기 ]
═══════════════════════════════════════════════════════════════════════
① 한 줄 요약
   "리셀 시장도 금융 시장처럼 미디어가 가격을 움직일까?" 를 왜 궁금해
   했는지 ─ 기존 연구들이 주식·암호화폐에서는 답을 줬는데 리셀에는 없다.

② 쉬운 설명
   주식 시장에서는 이미 답이 나와 있다:
     · Google 검색이 늘면 그 종목이 곧 오른다 (Da 2011)
     · 뉴스가 부정적이면 단기 수익률이 떨어진다 (Tetlock 2007)
     · 트위터 무드가 좋아지면 다우지수가 따라 움직인다 (Bollen 2011)
   그러면 스니커즈·카드·레고도 마찬가지일까? — 아무도 안 봤다.

③ 숫자의 의미
   본 슬라이드엔 수치 없음. 청중에게 '연구 공백'을 각인시키는 슬라이드.

④ 질문 받으면
   - "왜 하필 리셀이냐?" → 거래량은 작지만 정보 비대칭이 크고 '관심'에
     민감해서, 금융 시장 직관이 가장 잘 작동할 자산군.

[ S3. 연구 문제 RQ1·RQ2·RQ3 ]
═══════════════════════════════════════════════════════════════════════
① 한 줄 요약
   세 가지 질문으로 연구를 쪼갰다 — 인과성, 자산별 분화, 예측력.

② 쉬운 설명
   · RQ1 (선행성) — "미디어가 가격을 앞서가는가?"
     → 통계학에서 'A가 B를 선행한다'를 확인하는 도구가 Granger 검정.
   · RQ2 (자산별 분화) — "스니커즈·카드·레고가 같은 채널의 영향을 받나?"
     → 자산마다 유의했던 채널 목록을 만들어서 겹침을 측정.
   · RQ3 (예측력) — "통계로 골라낸 채널만 써도 예측이 잘 되나?"
     → 머신러닝(XGBoost)로 비교.
   세 가지가 점점 깊이 들어가는 구조: 있는가 → 다른가 → 쓸 만한가.

③ 숫자의 의미
   3 자산 × 5 채널 = 15회 (RQ1에서 실제로 돌릴 검정 횟수).

④ 질문 받으면
   - "왜 세 자산만?" → 데이터 접근성 + 시장 성숙도. 명품·시계도 가능하나
     향후 연구로 미뤘다.
   - "Granger가 '진짜' 인과성인가?" → 아니다, '시간 순서상 선행'을 본다.
     진정한 인과성과는 다른 개념. 한계로 명시.

[ S5. 미디어 채널 5개 ]
═══════════════════════════════════════════════════════════════════════
① 한 줄 요약
   '미디어'를 어떻게 숫자로 바꿨는지 ─ 다섯 종류의 채널 정의.

② 쉬운 설명
   · CH1 Google Trends = "사람들이 얼마나 검색했나" (관심도)
   · CH2 GDELT 뉴스량  = "뉴스에 얼마나 자주 나왔나" (보도 빈도)
   · CH3 GDELT 톤      = "뉴스 어조가 긍정/부정/중립이었나" (감성)
   · CH4 YouTube 조회수 = "유튜브에서 얼마나 봤나" (영상 관심)
   · CH5 댓글 감성     = "유튜브 댓글이 긍정인가 부정인가" (대중 반응)

   왜 그냥 안 쓰고 변환하나? → 단위가 다 다르다. 검색은 0~100,
   조회수는 수만~수억. 그대로 비교하면 큰 숫자가 항상 이긴다.
   그래서 z-score 등으로 '같은 잣대'로 만든다.

③ 숫자의 의미
   CH3가 빨간 박스인 이유 = 원래 NewsAPI + FinBERT (도메인 특화) 쓰려
   했는데, NewsAPI 무료는 과거 30일만 줘서 4년치 불가능. GDELT 톤
   (일반 감성)으로 대체. 한계 솔직히 인정.

④ 질문 받으면
   - "왜 트위터 안 썼나?" → API 정책 변경으로 역사 수집이 비싸졌다.
   - "FinBERT가 뭐냐?" → 금융 도메인 텍스트로 사전학습한 BERT 모델.
     CH5 댓글 감성에 사용.

[ S6. 분석 파이프라인 ]
═══════════════════════════════════════════════════════════════════════
① 한 줄 요약
   원시 데이터부터 결론까지의 8단계 흐름. "이 순서로 했다."

② 쉬운 설명
   01 수집 → 02 점수 변환 → 03 통합 → 04 ADF (정상성)
        → 05 Granger → 06 Jaccard → 07 XGBoost+SHAP → 08 DM
   요리에 비유하면:
     01~03 = 재료 사고 손질하기
     04   = 재료 상태 점검 (상했는지)
     05~06 = 메인 요리 (RQ1·RQ2)
     07~08 = 디저트/검증 (RQ3)

③ 숫자의 의미
   "원시 → Granger 직접 투입 금지" = 단위 안 맞춘 채로 통계 돌리면
   결과가 망가짐. 무조건 점수 변환 후.

④ 질문 받으면
   - "왜 ADF 먼저?" → 시계열이 '정상성(stationary)'이라야 Granger가
     유효. 트렌드가 있으면 차분(diff) 후 사용. ⇒ S7 정상성 검정에서 다룸.

[ S7. RQ1 Granger 결과 ]
═══════════════════════════════════════════════════════════════════════
① 한 줄 요약
   "어떤 채널이 어떤 자산을 앞서가는가" 를 15번 검정한 결과 표.

② 쉬운 설명
   Granger 검정 = "A의 과거값을 알면 B의 미래값을 더 잘 맞출 수 있는가?"
   를 통계적으로 검정.
     · F 통계량 = '효과 크기'. 클수록 선행성이 강함.
     · p 값    = '우연일 확률'. 작을수록 우연이 아니다 (=유의).
       관행상 p<0.05이면 'SIGNIFICANT'.
     · lag    = "몇 개월 앞서가는지" (1이면 1개월 전).

   직관: "이번 달 CH3 뉴스 감성이 좋으면, 다음 달 스니커즈 가격이
   오를 가능성이 통계적으로 입증되는가?" 같은 질문.

③ 숫자의 의미
   · sneakers CH3: F=13.38, p=0.001 → '13배 정도 효과 크고, 우연 확률
     0.1%'. 가장 강한 발견.
   · cards CH1: F=8.42, p=0.006 → 카드는 검색 트렌드만 깔끔히 선행.
   · lego CH1: p=0.054 → 살짝 모자라 'MARGINAL'. 진짜 신호인지 애매.
   · 나머지 10개 → 비유의 (영향 없음).

   BH 보정: "여러 번 검정하면 우연히 유의해 보이는 게 섞임. 그래서
   보정해서 진짜 살아남는 것만 보면" 4 → 2개 (sneakers CH3, cards CH1)
   만 남음.

④ 질문 받으면
   - "왜 F<2면 비유의로 보는가?" → 본 연구는 p<0.05 단일 기준이고,
     F는 효과 크기 참고용. F 자체로 cutoff는 안 둔다.
   - "왜 BH 보정 적용을 본문 아닌 부록에 두나?" → 응용 계량 경제 문헌
     관행 + 본 연구는 '선별 도구'라 너무 보수적이면 정보 손실. 그러나
     견고성 확인 위해 부록 병기.

[ S8. Lag 견고성 ]
═══════════════════════════════════════════════════════════════════════
① 한 줄 요약
   "lag(몇 개월 전) 을 어떻게 정하느냐에 결과가 흔들리는가" 점검 표.

② 쉬운 설명
   원래 분석은 BIC 라는 정보량 기준으로 lag 1~4 중 '최적' lag 하나만
   골랐다. 그런데 그 하나가 운 좋게 유의한 거였다면? → 1, 2, 3개월
   세 lag에서 다 돌려봐서 '여러 lag에서 일관되게 유의한지' 본다.

   ✓ (체크) = 2개 이상 lag에서 유의 → 진짜 견고함.
   △ (세모) = 1개 lag에서만 유의 → 의심스러움.

③ 숫자의 의미
   · sneakers CH3는 1·2·3 모두에서 p<0.01 → 가장 견고.
   · sneakers CH5는 BIC 최적(L=1)에서는 비유의(p=0.764)였지만,
     L=2, 3에서 p=0.016, 0.009로 유의 → '댓글 감성은 2~3개월 늦게
     가격에 반영된다'는 새로운 가설을 만들어 줌. (덤으로 얻은 발견)

④ 질문 받으면
   - "BIC 가 뭐냐?" → 베이지안 정보 기준. 모형이 '얼마나 데이터를 잘
     설명하면서도 복잡하지 않은가'를 측정. 작을수록 좋음.

[ S9. RQ2 Jaccard ]                ★ 발표자 추가 학습 필요 ★
═══════════════════════════════════════════════════════════════════════
① 한 줄 요약
   "자산마다 가격을 선행하는 채널 목록이 다른가" 를 정량적으로 비교.

② 쉬운 설명
   Jaccard 유사도 = 두 '집합'이 얼마나 겹치는지를 0~1로 보는 지표.
     공식:  J(A, B) = |A ∩ B| / |A ∪ B|
       · A ∩ B = 교집합 (둘 다에 들어있는 원소)
       · A ∪ B = 합집합 (둘 중 어디라도 들어있는 원소)

   예시로 직관 얻기:
     A = {사과, 배, 감}, B = {사과, 배, 포도}
     교집합 = {사과, 배} → 2개
     합집합 = {사과, 배, 감, 포도} → 4개
     J = 2/4 = 0.5

   · J=1 : 두 집합이 완전히 같음
   · J=0 : 한 원소도 안 겹침
   · 본 연구 임계:
       J<0.3       → '구성 다름' (RQ2 지지)
       0.3≤J≤0.6  → '부분적 차이'
       J>0.6       → '구성 유사' → 결론 수정

   본 연구의 '집합' = 그 자산에서 SIGNIFICANT 판정받은 채널들의 모음.

③ 숫자의 의미
   각 자산의 유의 채널 집합:
     sneakers = {CH1, CH3, CH4} (3개)
     cards    = {CH1}            (1개)
     lego     = ∅ (없음)         (0개)

   자산쌍 비교:
   · sneakers vs cards
       교집합 = {CH1} (둘 다 가짐) → 1개
       합집합 = {CH1, CH3, CH4} → 3개
       J = 1/3 = 0.333 → '부분적 차이' 구간
       의미: 둘 다 검색 트렌드는 공유하지만 sneakers는 뉴스·유튜브도
       추가로 가짐.

   · sneakers vs lego
       lego가 공집합 → 교집합 = 0개 / 합집합 = 3개
       J = 0/3 = 0.000 → 완전히 다름

   · cards vs lego
       J = 0/1 = 0.000 → 완전히 다름

④ 발표 중 청중이 물을 만한 것
   - "공집합인 lego와 다른 자산을 비교하는 게 의미 있나?"
     → 좋은 지적. 'lego는 유의 채널이 없다'는 사실 자체가 결과의 일부.
        J=0이 의미하는 건 '레고는 다른 자산과 다른 정보 처리 메커니즘
        을 가질 가능성' — 한계 절에 보완.
   - "왜 MARGINAL은 집합에 안 넣었나?"
     → 본 절에서는 보수적으로 SIGNIFICANT만 사용. Model B(예측 모형)
        에서는 MARGINAL도 포함하는 다른 기준 적용.
   - "0.333이면 부분적 차이라고 하는데, 0.3 컷오프는 자의적 아닌가?"
     → 솔직히 자의적임. 절대 기준 없음. 본 연구는 방법론 절에서 임계를
        사전 명시하고 일관 적용 (post-hoc 변경 없음).

⑤ 발표자가 기억할 핵심 한 줄
   "RQ2의 답은 — 자산마다 가격을 움직이는 미디어가 다르다. 단,
   검색 트렌드(CH1)는 sneakers와 cards가 공유하는 공통 신호다."

[ S10. RQ3 XGBoost + DM ]          ★ 발표자 추가 학습 필요 ★
═══════════════════════════════════════════════════════════════════════
① 한 줄 요약
   "Granger로 선별한 채널만 써도 예측이 잘 되는가" 를 머신러닝으로 검증.

② 쉬운 설명
   여기서 처음 등장하는 개념 세 가지를 풀어 정리:

   (1) XGBoost
       - '여러 결정 트리(decision tree)를 차곡차곡 쌓아 만든 모델'.
         표 형식 데이터에서 가장 잘 쓰는 머신러닝 알고리즘.
       - 여기선 '이번 달 채널 값들을 보고 다음 달 가격이 오를지 떨어질지'
         를 예측.

   (2) TimeSeriesSplit
       - 시계열은 미래를 보고 과거를 학습시키면 안 된다(cheating).
         그래서 '시간 순서대로 잘라서' 학습-검증을 함.
       - 4 splits, test_size=6 = 마지막 24개월을 검증으로 활용.

   (3) Diebold–Mariano (DM) 검정
       - 두 예측 모형(A, B)이 만들어내는 '시점별 오차(error)' 시계열을
         비교해서 '두 모형이 통계적으로 정말 다른지' 검정.
       - 직관: A의 오차들 - B의 오차들 = 차이 시계열. 이 차이가 평균적
         으로 0인지(=동등), 아닌지(=차이 있음)를 본다.
       - DM 통계량 부호:
           DM < 0 → Model A의 오차가 더 작음 (A 우수)
           DM > 0 → Model B의 오차가 더 작음 (B 우수)
       - p 값:
           p<0.05 → 차이 있음
           p>0.10 → 사실상 차이 없음 (등가)
           0.05~0.10 → 경계
       - 회귀(가격 수준) 기준으로 측정.

③ 숫자의 의미
   · '11/15 = 73%' = 다수 아이템에서 A와 B가 동등 → "전 채널 안 써도
     큰 손해 없다"
   · sneakers_jordan1 DM=-4.29 → Model A 압도적 우수 (전 채널이 더 좋음)
   · lego_falcon DM=-2.28, p=.032 → 역시 A 우수
   · RMSE 평균:
     - sneakers: A 452 < B 569 (A가 117 낮음)
     - lego:     A 176 < B 199 (A가 23 낮음)
     - cards:    A 334 ≈ B 329 (사실상 동등)
     → "평균 RMSE는 A가 낮은데 DM은 차이 없다고 한다"는 모순적 결과 →
        절제된 해석 필요.

   왜 모순처럼 보이나? — DM은 시점별 부호 차이를 자기상관 보정 후
   본다. 평균 RMSE는 절대 오차의 산술 평균. 큰 outlier 한두 개에
   영향받는 정도가 다름.

④ 발표 중 청중이 물을 만한 것
   - "왜 분류(0/1)는 안 보고 회귀로 DM 했나?"
     → DM은 원래 점예측(point forecast) 오차 비교용. 분류 확률로 DM
        하면 결과 불안정. 두 태스크를 보완 지표로 같이 보고 (한계 명시).
   - "73%가 동등이면 B가 더 낫다고 할 수 있는 거 아닌가?"
     → 아니다. 'B가 A보다 더 좋다' (B 우수)인 경우는 1건뿐. 'A가 B보다
        더 좋다' 인 경우는 2건. 그래서 '동등 또는 A 약간 유리'로 절제.
   - "왜 jordan1과 falcon만 A가 압도적인가?"
     → SHAP 분석 결과 보면, 이 둘은 Granger 선별이 빠뜨린 채널(CH5,
        CH4)이 실제로는 핵심 기여를 함. Model B가 정보 손실.

⑤ 발표자가 기억할 핵심 한 줄
   "RQ3의 답은 — Model B가 우수하다는 결론은 못 내림. 하지만 73%
   아이템에서 'A와 동등'이니, '큰 손해 없이 채널 수를 줄일 수 있는
   도구일 수 있다'는 절제된 해석은 지지된다."

[ S12. SHAP vs Granger 표 ]
═══════════════════════════════════════════════════════════════════════
① 한 줄 요약
   "통계적으로 선행하는 채널" (Granger) 과 "예측에 실제로 기여하는
   채널" (SHAP) 의 순위가 다르다는 점을 보여주는 표.

② 쉬운 설명
   SHAP = "이 변수가 모델 예측에 얼마나 기여했는가" 를 변수별로 분해해
   주는 도구. (게임이론의 Shapley value 기반)
   |SHAP| 평균 = 그 변수의 평균적 기여도. 클수록 중요.

   Granger와 SHAP의 차이:
     · Granger : '변수 하나만 단독으로' 가격 라그만 통제하고 선형으로
       선행하는지 본다.
     · SHAP    : '모든 변수가 다 같이 들어간' 비선형 트리 모형 안에서,
       그 변수의 '한계 기여도'를 본다.

   왜 두 결과가 어긋날까?
     - 가격 모멘텀 변수(price_vs_ma3 등)가 미래 가격 분산의 대부분을
       흡수해 버리면, 선형 선행 채널이라 해도 SHAP 기여는 작아짐.
     - 즉 "이미 가격이 자기 자신의 과거값으로 다 예측되니까, 채널 추가
       기여가 작아 보이는" 현상.

③ 숫자의 의미
   sneakers를 예로:
     · CH3 |SHAP|=0.011 (5등) ← Granger F 1위(SIG)
     · CH5 |SHAP|=0.124 (2등) ← Granger 비유의(NOT)
   이 둘이 완전히 뒤집힘 → 가장 극적인 사례.

④ 질문 받으면
   - "그럼 어느 게 맞는 거냐?" → 둘 다 맞다. 측정 대상이 다른 보완
     지표. 둘을 같이 봐야 채널의 역할을 입체적으로 이해할 수 있다.
   - "이게 본 연구의 한계인가 기여인가?" → 기여. 인과성과 예측력을
     '분리해서' 평가했다는 것이 방법론 기여.

═══════════════════════════════════════════════════════════════════════
 ★ 단어장 (모르겠으면 여기서 빨리 확인)
═══════════════════════════════════════════════════════════════════════

· Granger 인과성
    A의 과거값이 B의 미래값 예측에 추가 정보를 주면 'A가 B를 Granger
    선행'한다고 함. '시간 순서상 선행'이며 진정한 인과성과 다름.

· F 통계량 / p 값
    F = 두 모형 (제한 vs 비제한)의 잔차 비율. 클수록 비제한 모형이
    유의미하게 더 잘 맞음. p = 'F가 우연으로 그 정도 큰 값이 나올 확률'.

· lag (래그)
    '몇 시점 전 값을 사용할 것인가'. lag=1이면 1개월 전, lag=2면
    2개월 전.

· BIC (베이지안 정보 기준)
    모형의 적합도를 '복잡도 패널티' 와 함께 보는 지표. 작을수록 좋은
    모형. lag 선택에 사용.

· ADF 검정 (Augmented Dickey-Fuller)
    시계열이 '정상성(stationary)' 인지 검정. p<0.05면 정상.
    비정상이면 차분(현재값 - 과거값)을 취해 정상화.

· Jaccard 유사도
    두 집합의 교집합을 합집합으로 나눈 값. 0~1 사이. 1이면 같은 집합.

· Diebold–Mariano (DM) 검정
    두 예측 모형의 오차 시계열을 비교해 통계적 차이 검정. p<0.05면
    유의한 차이.

· RMSE (Root Mean Squared Error)
    예측 오차의 제곱 평균의 제곱근. 작을수록 정확. 단위 = 예측 대상의
    단위.

· SHAP value
    한 예측값에 각 변수가 얼마나 기여했는지를 게임이론(Shapley value)
    으로 분해. |SHAP| 평균 = 변수의 '전역 중요도'.

· z-score
    (값 - 평균) / 표준편차. 평균 0, 표준편차 1로 표준화. 단위 다른
    변수 비교 가능.

· BH 보정 (Benjamini-Hochberg FDR)
    다중 검정에서 '거짓 양성률(FDR)' 통제. 여러 번 검정할수록 우연
    유의가 늘기 때문에 보수적으로 임계 조정.

· FinBERT
    금융 텍스트로 사전학습한 BERT. P(긍정), P(부정), P(중립) 산출.
    본 연구는 CH5 댓글 감성에 사용.

· XGBoost
    Gradient Boosting 트리 모형. 표 형식 데이터 SOTA. 분류·회귀 둘 다.

· TimeSeriesSplit
    시계열 교차검증. 시간 순서를 지켜서 학습-검증 분할 (미래→과거
    학습 금지).
